# dotfiles

個人環境向けの設定ファイルや補助スクリプトを、用途ごとに整理して管理する
設定を一箇所に集約するだけでなく、役割ごとに分割して見通しよく保守する

## 目的

- 各種ツールの設定を用途別に整理して管理する
- 初期セットアップやインストール作業をスクリプト化して再現性を高める
- 設定追加時に置き場所が迷子にならない、拡張しやすい構成を維持する

## 設計方針

- 設定は「ツール単位」または「役割単位」でディレクトリを分ける
- シェル設定は 1 ファイルに集約せず、関心ごとごとに分割する
- 再利用する処理は `scripts/` に寄せて共通化する
- 個人情報や秘匿情報はリポジトリに含めない

## ディレクトリ構成

このリポジトリは以下の構成で管理する

- `./scripts/**`
  - セットアップやインストールなど、複数箇所から利用する共通スクリプトを配置する
- `./zsh/**`
  - `zshrc` 関連の設定を項目ごとに分割して格納する
  - 例: `aliases.zsh`, `path.zsh`, `exports.zsh`
- `./**`
  - 各ツールや各項目の設定は、名前に対応するディレクトリを切って管理する
  - 例: `./vim/`, `./git/`, `./yarn/`

## 配置ルール

- Git 関連の設定は `./git/` にまとめる
- Git に関するエイリアスを追加する場合は `./git/aliases.zsh` のように、対象が明確な場所へ置く
- Zsh の設定は `./zsh/` 配下で機能ごとに分割し、1 ファイル 1 役割を意識する
- 新しいツール設定を追加する場合は、トップレベルに対応するディレクトリを作成して管理する

## スクリプト

環境構築や導入作業は手作業に寄せすぎず、スクリプトとして管理する

- `./scripts/bootstrap.sh`
  - 初期セットアップを行うための起動スクリプト
  - 新しい環境で最初に実行する入口とする
- `./scripts/install.sh`
  - 各種ツールや依存関係のインストールを担うスクリプト

- スクリプトは責務を明確に分け、必要に応じて OS 別や用途別に分割して管理する

## 運用ルール

- シークレットキー、トークン、秘密鍵などの秘匿情報はコミットしない
- メールアドレス、ユーザー名、端末固有の値など個人情報は極力含めない
- 環境依存の値が必要な場合は、直接書き込まず外部ファイルや環境変数で扱える形を優先する
- どこに置くべきか迷う設定は、まず「どのツールに属するか」を基準に配置先を決める

## 拡張方針

- ツールごとの導入手順を README から辿れるようにする
- セットアップ手順をスクリプトとドキュメントの両面で整備する
- 設定ファイルの配置ポリシーを明文化し、追加時の判断コストを下げる
